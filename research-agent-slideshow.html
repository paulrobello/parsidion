Change 2: Fix NotebookLM Syntax
The agent used Task(...) pseudo-code that is not valid Claude Code syntax. The Agent tool is invoked via the Agent tool call in tool use — not as inline text in an agent definition.
Before ❌ — invalid pseudo-code
Spawn a background subagent: ``` Task( prompt="Wait for deep research in notebook [id]... Use: notebooklm research wait -n [id]...", subagent_type="general-purpose" ) ```
After ✓ — correct guidance
Dispatch a general-purpose subagent with this prompt, then continue to step 3: > "Wait for deep research in > notebook [id] and import > all sources. > Use: notebooklm research > wait -n [id] --import-all"
The correct pattern: describe the prompt as a blockquote — the agent invokes the Agent tool itself. No pseudo-code syntax in agent definitions.
Change 3: agentchrome for Content Extraction
"Use the Web Fetch tool" can't render JS-heavy documentation sites like MDN or framework docs. agentchrome launches a headless Chrome instance, navigates to the URL, and extracts fully-rendered page text.
Standard extraction pattern
# Connect once per research session (launch headless Chrome) agentchrome connect --launch --headless # Navigate and extract visible text agentchrome navigate "https://example.com/docs" --wait-until networkidle agentchrome page text # not page snapshot — returns readable content
Before: "Web Fetch tool" → HTML dump, no JS rendering → MDN, Next.js docs: broken
After: agentchrome → full JS-rendered text → Fallback: Web Fetch if connect fails
--wait-until networkidle ensures single-page apps fully load before extraction
Change 4: Brave Search Rate-Limit Fallback
Brave Search hits rate limits regularly during heavy research sessions. Previously the agent had no guidance — it would silently fail or loop. CLAUDE.md already prescribes mcpl search as the fallback for rate-limited tools.
New "Handling Edge Cases" item 5
5. **Brave Search Rate Limits**: If the Brave Search tool returns a rate-limit error or fails to respond: - Run `mcpl search "search" --limit 5` to discover available alternative search MCP tools - Use the best available alternative to continue research - Note which search tool was used in the Summary Report if it differs from Brave Search
mcpl search "search" discovers live alternatives no hardcoded tool names
✓ Aligned with CLAUDE.md: "If Brave hits rate limits, check MCP Launchpad"
