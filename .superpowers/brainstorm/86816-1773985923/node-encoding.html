Node Visual Encoding

What do node color and size communicate?

🎨 Node Color

C1

By Folder

Each vault folder gets a distinct hue. Patterns=blue, Debugging=red, Research=green, Daily=gray…

C2

By Note Type

pattern / debugging / research / project / daily / tool / language / framework — 8 types, 8 colors

C3

By Confidence

Green=high, Yellow=medium, Red=low. Useful for spotting weak/stale notes in the graph.

⬤ Node Size

S1

Incoming Links

Bigger = more notes reference this one. Hub notes emerge visually — like PageRank for your vault.

S2

Graph Degree

Bigger = more semantic neighbors above threshold. Shows which notes are most "connected" conceptually.

S3

Uniform

All nodes the same size. Cleaner look, lets layout and color do the work.

My recommendation: Color by Folder (9 distinct clusters, maps to your mental model) + Size by Incoming Links (surfaces hub notes — instantly shows the most-referenced knowledge). You can always add a toggle to switch color dimension later.
