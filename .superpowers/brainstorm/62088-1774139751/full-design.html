Architecture Patterns

#architecture #vault #python

The vault uses a layered architecture with four main components: hook scripts, summarizer, shared library, and the vault filesystem itself.

Key Components

  • Hook scripts handle lifecycle events (session start/stop, compaction)
  • vault_common.py provides shared utilities
  • The summarizer processes queued sessions into vault notes

Architecture Patterns

Design Summary

Layout

  • Resizable + collapsible left sidebar with file tree
  • Sidebar filter box for quick tree filtering
  • Tab bar for multiple open notes
  • Full-width content area (read or graph)
  • Read/Graph toggle in top-right toolbar
  • Global ⌘K search in toolbar

Behavior

  • Click wikilink → replaces current tab
  • Cmd+click wikilink → new tab
  • Graph defaults to local neighborhood (2 hops)
  • Config option to default to full vault
  • "Show Full Vault" button in graph mode
  • Draggable HUD panel for graph controls
  • All settings persisted to localStorage
